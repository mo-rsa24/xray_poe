# 📝 Documentation

## Mission
Continuously document the results and progress of every phase — data, EDA, VAE,
LDM, metrics, composition — into a coherent running record that captures
decisions, figures, metrics, and gate outcomes as they happen, so the paper
write-up is assembly rather than reconstruction.

## Objectives
1. Maintain a running progress log per phase (what was done, decisions, gate outcomes).
2. Collect and caption the key figures and metrics from each scope.
3. Record each experiment's result against the pre-registration table (no post-hoc narration).
4. Assemble the material into paper-ready sections (methods, results) as phases complete.

## Goals
1. Per-phase progress documented as phases complete.
2. Figures + metrics collected and captioned.
3. Pre-registration outcomes filed per experiment.
4. Draft methods / results sections assembled.

## Expected Outcome
A living document mirroring the project's progress — every phase's decisions,
figures, and gate outcomes recorded — so writing the paper is assembly, not
reconstruction.

## Definition of Done
1. Progress log exists with an entry per completed phase.
2. Figure + metric register maintained (source scope → figure/number → caption).
3. Each experiment's outcome filed against the pre-registration table.
4. Methods + results draft sections assembled from the register.
